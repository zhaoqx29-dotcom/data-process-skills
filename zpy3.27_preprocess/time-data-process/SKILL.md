# time_data_process - 数据预处理

主 skill，负责协调数据预处理全流程。

## 触发条件

用户描述包含以下任一场景时触发：

任何数据预处理相关的需求：
- **数据加载、读取、查看**："加载数据"、"读取数据"、"查看数据"、"导入数据"
- **数据质量评估、检查**："数据评估"、"数据质量检查"、"查看缺失值"、"数据检查"、"数据概况"
- **数据清洗**："清洗数据"、"数据预处理"
- **缺失值处理**："处理缺失值"、"处理空值"、"缺失值处理"、"填充缺失值"
- **滤波降噪**："滤波"、"去噪"、"降噪"、"平滑数据"、"数据平滑"
- **标准化归一化**："标准化"、"归一化"、"Min-Max"、"缩放"、"对数变换"
- 其他涉及结构化表格数据预处理的需求

## Agent 工作流程

### 阶段一：数据加载与业务语义理解

```
1. 询问数据文件路径（如果未提供）
   └── 用户未提供 → "请提供数据文件路径（支持 CSV、XLSX、XLS 格式）"
   └── 可指定文件编码，如 "encoding='gbk'" 或 "encoding='utf-8'"

2. 使用 analyze 子技能加载数据
   └── 调用 analyze 的 load_file(path, encoding=...) 智能加载数据
   └── 如果加载失败（如编码问题），尝试常见编码：'gbk', 'gb2312', 'utf-8-sig'

3. 询问用户业务语义（关键步骤，必须执行）
   └── "请描述一下这个数据的业务场景（如销售预测、用户分析、库存管理等）"

4. 询问用户选择处理模式
   └── "请选择处理模式："
   └── [1] 手动输入 - 用户指定处理列和需求
   └── [2] 自动处理 - skill 自动诊断数据并处理
   └── 用户输入 1 或 2

   └── 如果选择 [1] 手动输入：
       └── 询问："需要处理哪些数据列及处理需求？请按格式输入：列名+处理类型，如FE_y:缺失值填充,滤波降噪."
       └── 支持同一列选择多种处理类型
       └── 支持输入多个列，用分号或换行分隔
       └── 用户自由输入列名和需求
       └── 跳到阶段二继续执行

   └── 如果选择 [2] 自动处理：
       └── 调用 process 的 interactive_confirm_and_process(df)
       └── 自动进行数据诊断、打印报告、交互确认、处理
       └── 跳到阶段四执行并生成报告
```

### 阶段二：数据分析与处理推荐（仅手动输入模式）

```
仅当用户在阶段一选择 [1] 手动输入 时执行此阶段。

优先使用【精炼优化任务定义-数据预处理建议.md】中的处理方式

4. 使用 analyze 子技能进行数据分析
   └── 调用 analyze 的 assess() 评估数据质量
   └── 仅针对用户指定需要处理的列进行分析
   └── 输出：df, assess_result, analysis_report

5. 生成文字版数据分析报告（必须执行，仅展示用户指定处理列）
   └── 报告格式：📊 数据分析报告
   └── 包含数据概览、各列详细信息、数据质量评估
   └── 使用 analyze 的 generate_analysis_report() 生成

6. 使用 recommend 子技能生成推荐策略
   └── 调用 recommend 的 parse_target_columns() 解析用户输入
   └── 调用 recommend 的 recommend_strategy() 生成推荐策略
   └── 调用 recommend 的 generate_strategy_table() 生成策略表
   └── 调用 recommend 的 generate_plan() 生成调用链表
   └── 输出：strategy_table, plan

7. 【新增】询问用户是否需要自定义处理策略
   └── [n] 使用自动推荐方法，继续步骤8
   └── [Y] 进入自定义策略流程：
       a. 选择列和处理类型
       b. 选择策略类型：
          - 预定义方法（从脚本方法中选择）
          - 自定义功能组合（如"取A、B、C三列的众数填入D列"）
          - 填充指定值（如 0、"未知"）
          - 自定义计算表达式（如"用列1的均值填充列2"）
          - 跳过处理（该列此类型不处理）
       c. 收集具体参数
       d. 询问是否还需为其他列自定义策略

8. 按处理类型优先级生成初始执行顺序
   └── 缺失值处理 → 滤波降噪 → 标准化归一化
   └── 同类型内的处理按列分组执行
```

### 阶段三：处理计划展示与确认（仅手动输入模式）

```
仅当用户在阶段一选择 [1] 手动输入 时执行此阶段。

9. 展示完整处理任务表
   └── 格式：| 列名 | 处理类型 | 推荐方法 | 备选方法 | 推荐理由 |

10. 展示初始执行顺序（按处理类型优先级排序）
    └── 步骤1：缺失值处理（列出各列及方法）
    └── 步骤2：滤波降噪（列出各列及方法）
    └── 步骤3：标准化归一化（列出各列及方法）

11. 询问：是否需要修改执行顺序或具体方法？[Y/n]
    └── [n] 直接进入阶段四执行
    └── [Y] 进入修改流程：
        a. 允许用户调整步骤顺序（仅限于处理类型级别）
        b. 允许用户更改每列的具体处理方法
        c. 确认后进入阶段四
```

### 阶段四：执行处理与结果评价

```
12. 构建 chain_execute 所需的 plan 列表
    └── 根据确认后的处理计划，按步骤顺序构建 plan

13. 使用 process 子技能执行数据处理
    └── 调用 process 的 chain_execute() 执行链式处理
    └── df_result = chain_execute(df, plan, verbose=True)
    └── 输出：df_result

14. 使用 process 子技能生成评价报告
    └── 调用 process 的 generate_evaluation_report() 生成处理结果评价报告
    └── 输出：evaluation_report

15. 使用 process 子技能生成对比图表
    └── 调用 process 的 generate_comparison_chart() 生成对比图表
    └── 图表保存为：数据处理前后图表对比.png 或 .jpg
    └── 输出：chart
```

## 业务语义询问清单

必须询问用户的信息包括：
1. **业务场景**：这份数据是做什么用的？（销售预测、用户分析、库存管理等）
2. **处理列及需求**：需要处理哪些列和进行什么处理？（用户自由输入，格式如：FE_y:缺失值填充,滤波降噪；计划重量:标准化归一化）

**输入格式说明：**
- 支持格式：`列名:处理类型1,处理类型2` 或 `列名 处理类型1 处理类型2`
- 多个列用分号或换行分隔
- 处理类型可选：缺失值填充、滤波降噪、标准化归一化
- 支持同一列选择多种处理类型

**输入示例：**
```
FE_y:缺失值填充,滤波降噪
实际重量:缺失值填充,标准化归一化
计划重量:滤波降噪
```

## 自动策略推荐规则

### 缺失值处理推荐
| 数据特征 | 推荐方法 | 说明 |
|---------|----------|------|
| 缺失值出现在数据开头 + 缺失块较短（≤5行） | fill_missing_bfill | 后向填充，适合数据前段缺失场景 |
| 类别型数据 + 众数占比超过40% | fill_missing_mode | 众数填充，使用最频繁值 |
| 连续少量缺失（≤3行） + 随时间缓慢变化/时间序列 | fill_missing_ffill | 前向填充，保留时间趋势 |
| 连续多行缺失（>3行） + 明显线性/平滑趋势 | fill_missing_interpolate | 线性插值，适合多行连续缺失 |
| 数值型 + 存在显著异常值 | fill_missing_median | 中位数填充，对离群值不敏感 |
| 数值型 + 偏态分布 | fill_missing_median | 中位数填充，适合偏态分布 |
| 数值型 + 正态分布 | fill_missing_mean | 均值填充，适合正态分布 |
| 缺失率 > 30% | 询问用户 | 建议谨慎处理，可能需要业务确认 |

**缺失值处理推荐优先级（由高到低）：**

1. **后向填充 (bfill)**：缺失值出现在数据开头 + 缺失块较短（≤5行）
2. **众数填充 (mode)**：类别型数据 + 众数占比超过40%
3. **前向填充 (ffill)**：连续少量缺失（≤3行）+ 随时间缓慢变化/时间序列数据
4. **线性插值 (interpolate)**：连续多行缺失（>3行）+ 明显的线性或平滑趋势
5. **中位数填充 (median)**：数值型 + 存在显著异常值/偏态分布
6. **均值填充 (mean)**：数值型 + 正态分布

**详细说明：**
- **fill_missing_bfill**：当检测到缺失值出现在数据开头（前10%）且缺失块较短（≤5行）时推荐，使用后向填充
- **fill_missing_mode**：类别型数据优先使用众数填充，当众数占比超过40%时更加确信
- **fill_missing_ffill**：当缺失值为连续少量缺失（最大连续缺失≤3行）且数据有随时间缓慢变化特征或为时间序列数据时推荐
- **fill_missing_interpolate**：当存在连续多行缺失（最大连续缺失>3行）且数据显示明显的线性或平滑趋势时推荐
- **fill_missing_median**：当数值型数据存在显著异常值（IQR方法检测，离群点比例>5%）或呈偏态分布（偏度绝对值>1）时推荐
- **fill_missing_mean**：当数值型数据接近正态分布（偏度绝对值<0.3）时推荐使用均值填充

### 滤波降噪推荐
| 数据特征 | 推荐方法 | 说明 |
|---------|----------|------|
| 存在显著异常值（孤立尖峰、脉冲噪声） | filter_median | 中值滤波专治孤立尖峰、脉冲噪声，适配离散数据，不模糊数据边缘 |
| 随机小幅波动噪声（中等波动） | filter_moving_avg | 抑制随机小幅波动噪声，适配平稳/周期性数据，窗口越大平滑越强 |
| 周期性强的数据（需从频率维度分离） | filter_fourier | 从频率维度分离噪声与信号，适配周期性强的数据 |
| 平稳数据 + 平滑趋势 | filter_moving_avg | 抑制随机小幅波动噪声，适配平稳/周期性数据 |
| 时间序列数据 + 周期性 | filter_fourier | 从频率维度分离噪声与信号，需先识别噪声对应的频率段 |

**滤波降噪推荐优先级（由高到低）：**

1. **中值滤波 (filter_median)**：存在显著异常值（孤立尖峰、脉冲噪声）
2. **移动平均滤波 (filter_moving_avg)**：随机小幅波动噪声（中等波动）或平稳数据+平滑趋势
3. **傅里叶变换滤波 (filter_fourier)**：周期性强的数据（需从频率维度分离噪声与信号）

**详细说明：**

**filter_median（中值滤波）- 核心适用特征：**
- 专治孤立尖峰、脉冲噪声：对异常值（离群点）非常有效，是非线性滤波器
- 适配离散数据：适合处理离散数据，不会引入虚假值
- 不模糊数据边缘：能够较好地保留数据边缘和突变特征
- 推荐条件：当检测到存在显著异常值（IQR方法检测，离群点比例>5%）时推荐使用中值滤波

**filter_moving_avg（移动平均滤波）- 核心适用特征：**
- 抑制随机小幅波动噪声：通过平均值平滑数据，减少随机抖动
- 适配平稳/周期性数据：适合处理平稳数据或具有明显周期性的数据
- 窗口越大平滑越强：窗口大小控制平滑程度，通常取3-20，大窗口平滑强但可能丢失细节
- 推荐条件：当检测到中等波动（变异系数0.1-0.3）或平稳数据+平滑趋势时推荐

**filter_fourier（傅里叶变换滤波）- 核心适用特征：**
- 从频率维度分离噪声/信号：通过频域分析识别噪声频率段，精准滤除
- 适配周期性强的数据：适合处理具有明显周期性结构的数据（如振动信号、音频）
- 需先识别噪声对应的频率段：需要分析频谱，确定噪声所在的频率范围
- 推荐条件：当数据具有强周期性特征（差分稳定性检测）或为时间序列数据时推荐

### 标准化归一化推荐
| 数据特征 | 推荐方法 | 说明 |
|---------|----------|------|
| 需消除量纲影响 | normalize_minmax | Min-Max 归一化到 [0,1] |
| 需要特定业务范围 | normalize_custom_range | 指定区间缩放（如评分0-100） |
| 右偏分布/长尾数据 | normalize_log | 对数变换使分布更接近正态 |

**推荐规则详情：**
- **normalize_minmax**: 当数值列量纲差异大（如一个列是0-1，另一个是0-10000）时推荐，默认缩放到 [0,1]
- **normalize_custom_range**: 当业务有特定范围要求时推荐（如温度摄氏度、热度值0-100分）
- **normalize_log**: 当偏度 > 1（右偏分布）或数据呈现长尾特征时推荐，自动处理非正数据（添加偏移量）

### 图表推荐规则

| 处理类型 | 推荐图表类型 | 图表说明 |
|----------|------------|----------|
| 缺失值处理 | 缺失值分布图（柱状图/条形图） | 展示各列缺失值数量和填充后的变化 |
| 缺失值处理 | 填充前后数据分布对比（直方图/箱线图） | 对比填充前后的数据分布情况 |
| 滤波降噪 | 原始与滤波后数据对比（折线图/趋势图） | 展示滤波前后的数据趋势变化，突出平滑效果 |
| 滤波降噪 | 谱分析图（如适用） | 展示频域分析，说明噪声成分的消除 |
| 标准化归一化 | 处理前后数据范围对比（直方图/箱线图） | 展示数据在新尺度下的分布情况 |
| 标准化归一化 | Before/After 对比图（散点图/QQ图） | 展示数据变换后的分布变化 |

**图表生成规则详情：**
- **缺失值填充图表**：使用柱状图展示填充前后缺失值数量的变化，使用直方图对比数据分布
- **滤波降噪图表**：使用折线图展示原始数据与滤波后数据的对比，突出标识被平滑的异常点
- **标准化归一化图表**：使用直方图或箱线图展示数据范围的变化，可视化量纲统一效果
- **图表保存格式**：默认保存为 PNG 格式，文件名为 "数据处理前后图表对比.png"
- **图表布局**：多子图布局，每个处理类型一个子图，清晰展示处理效果
- **normalize_minmax**: 当数值列量纲差异大（如一个列是0-1，另一个是0-10000）时推荐，默认缩放到 [0,1]
- **normalize_custom_range**: 当业务有特定范围要求时推荐（如温度摄氏度、热度值0-100分）
- **normalize_log**: 当偏度 > 1（右偏分布）或数据呈现长尾特征时推荐，自动处理非正数据（添加偏移量）

## 子 Skill 调用

### analyze
数据分析子 skill，负责数据加载、评估和生成报告。

**数据加载：**
- `load_file(path, **kwargs)` - 智能加载，根据扩展名自动识别
- `load_csv(path, encoding, **kwargs)` - 加载 CSV 文件
- `load_excel(path, sheet_name, **kwargs)` - 加载 Excel 文件

**数据评估：**
- `assess(df)` - 评估数据质量

**报告生成：**
- `generate_analysis_report(df, assess_result, target_columns)` - 生成分析报告

### recommend
策略推荐子 skill，负责推荐处理策略并生成调用链表。

**策略推荐：**
- `recommend_strategy(df, columns, business_context)` - 智能推荐处理策略

**策略表生成：**
- `generate_strategy_table(strategies, parsed_columns)` - 生成策略表
- `generate_plan(strategies, parsed_columns)` - 生成调用链表

### process
数据处理子 skill，负责执行数据处理、生成评价报告和对比图表。

**链式执行：**
- `chain_execute(df, plan, verbose)` - 链式执行多种处理操作

**缺失值处理：**
- `fill_missing_ffill(df, columns)` - 前向填充
- `fill_missing_bfill(df, columns)` - 后向填充
- `fill_missing_mean(df, columns)` - 均值填充
- `fill_missing_median(df, columns)` - 中位数填充
- `fill_missing_mode(df, columns)` - 众数填充
- `fill_missing_interpolate(df, columns)` - 线性插值

**滤波降噪：**
- `filter_median(df, columns, window_size)` - 中值滤波
- `filter_moving_avg(df, columns, window_size)` - 移动平均滤波
- `filter_fourier(df, columns, cutoff_freq)` - 傅里叶变换滤波

**标准化/归一化：**
- `normalize_minmax(df, columns, min_val, max_val)` - Min-Max 归一化
- `normalize_custom_range(df, columns, target_min, target_max)` - 指定区间缩放
- `normalize_log(df, columns, offset, base)` - 对数变换

**评价与图表：**
- `generate_evaluation_report(df_before, df_after, plan)` - 生成评价报告
- `generate_comparison_chart(df_before, df_after, plan)` - 生成对比图表

## Agent 注意事项

1. **先问业务语义再分析**：必须先了解业务场景，然后询问用户需要处理的列及处理需求
2. **用户自由输入列名和需求**：只询问一个问题"需要处理哪些数据列及处理需求？"，让用户自由输入"列名:处理类型1,处理类型2"格式
3. **解析用户输入**：正确解析用户输入的格式，支持多种分隔符（分号、换行、逗号）和冒号格式
4. **分析报告仅展示指定列**：数据分析报告只展示用户指定需要处理的列，不展示其他列
5. **区分自动推荐和用户自定义**：在策略展示时明确标注策略来源
6. **必须询问是否自定义**：在展示自动推荐后，必须询问用户是否需要自行定义策略
7. **处理结果要有评价**：最终报告不仅要展示执行结果，还要有效果评价和后续建议
8. **结合子 skill 脚本**：所有数据处理操作都通过调用 analyze、recommend、process 子技能的函数来实现
9. **必须保存报告**：数据分析报告和处理结果评价报告必须保存为 Markdown 文件
10. **必须生成并保存图表**：根据处理类型自动推荐合适的图表类型，生成处理前后对比图并保存为"数据处理前后图表对比.png"
