## 基于缺失率的策略选择

| 缺失率 | 建议策略 | 说明 |
|--------|----------|------|
| < 5% | 简单填充 | 直接使用统计量填充，影响较小 |
| 5%-15% | 统计填充 | 优先使用Median/Mode，对离群值不敏感 |
| 15%-30% | 慎重处理 | 需业务判断，可能需要模型填充或删除 |
| 30%-50% | 特殊处理 | 考虑删除该列或使用缺失指示变量 |
| > 50% | 不建议使用 | 列信息量不足，建议收集更多数据 |

## 填充方法选择指南

### 前向填充 (ffill)
- **适用场景**：时间序列数据
- **原理**：用前一个有效值填充
- **优点**：保留时间趋势
- **缺点**：首行无法填充

```python
# 适合：温度、湿度、股价等连续观测数据
df = fill_missing_ffill(df, ['temperature'])
```

### 后向填充 (bfill)
- **适用场景**：时间序列数据，且首行缺失
- **原理**：用后一个有效值填充
- **优点**：可填充首行
- **缺点**：可能违背时间因果性

```python
# 适合：时间序列首行缺失的情况
df = fill_missing_bfill(df, ['temperature'])
```

### 均值填充 (mean)
- **适用场景**：数值列，正态分布
- **原理**：用列的平均值填充
- **优点**：保持总体均值不变
- **缺点**：对离群值敏感，会低估方差

```python
# 适合：身高、体重、成绩等近似正态分布的数据
df = fill_missing_mean(df, ['score', 'height'])
```

### 中位数填充 (median)
- **适用场景**：数值列，偏态分布或存在离群值
- **原理**：用列的中位数填充
- **优点**：对离群值不敏感
- **缺点**：不适用于类别数据

```python
# 适合：收入、房价等偏态分布数据
df = fill_missing_median(df, ['income', 'price'])
```

### 众数填充 (mode)
- **适用场景**：类别列
- **原理**：用列中最频繁的值填充
- **优点**：保持数据分布
- **缺点**：可能引入偏差（当众数占比不高时）

```python
# 适合：城市、类别、状态等离散数据
df = fill_missing_mode(df, ['city', 'category', 'status'])
```

### 指定值填充
- **适用场景**：有特定业务含义的缺失
- **原理**：用预设的值（如"未知"、"0"、"NA"）填充
- **优点**：明确标记缺失状态
- **缺点**：可能影响某些算法

```python
# 适合：缺失本身有业务含义
df = fill_missing_value(df, 'city', '未知')
df = fill_missing_value(df, 'count', 0)
```

## 数据类型与填充方法映射

| 数据类型 | 推荐方法 | 备选方法 | 避免方法 |
|----------|----------|----------|----------|
| 时间序列 (datetime + 数值) | ffill / bfill | 插值法 | mean/median |
| 正态分布数值 | mean | median | mode |
| 偏态分布数值 | median | mean | mode/ffill |
| 类别/枚举 | mode | 指定"未知" | mean/median |
| 二元 (0/1) | mode | 指定值 | mean/median |
| 计数数据 (非负整数) | median | mode (0标记) | mean |

## 特殊场景处理

### 时间序列数据
```
1. 先检查是否为时间序列（有 date/time 列）
2. 按 date 排序
3. 优先使用 ffill 前向填充
4. 首行缺失使用 bfill 或线性插值
5. 不要使用 mean/median (会破坏时间趋势)
```

### 目标变量（Label）
```
1. 目标变量的缺失谨慎处理
2. 缺失率 < 5%：根据业务判断是否删除这些行
3. 缺失率 >= 5%：考虑不使用该条数据
4. 原则：不要填充目标列，宁缺毋滥
```

### 标识列 (ID)
```
1. ID 列缺失通常是数据质量问题
2. 生成新的 ID 序列重新赋值
3. 或使用 fill_missing_ffill() + 手动检查
```

## 填充效果评估

填充后应检查以下指标：

1. **数据分布变化**：填充前后均值、方差是否合理
2. **缺失值减少**：确保目标列的缺失值已填充
3. **相关性影响**：填充是否引入了不合理的关联

```python
# 填充前后对比
before = assess(df_before)
after = assess(df_after)

# 检查每列的缺失值减少情况
for col in df.columns:
    print(f"{col}: {before['missing'][col]['count']} -> {after['missing'][col]['count']}")
```

## 常见陷阱

| 陷阱 | 问题 | 解决方案 |
|------|------|----------|
| 时间序列用均值填充 | 破坏时间趋势 | 使用 ffill/bfill |
| 类别数据用均值填充 | 可能报错或产生奇怪值 | 使用 mode |
| 目标列也被填充 | 可能影响模型训练 | 不填充目标列 |
| 高缺失率列仍要填充 | 填充可信度低 | 考虑删除该列 |
| 填充后不检查 | 可能引入新问题 | 对比填充前后统计量 |
